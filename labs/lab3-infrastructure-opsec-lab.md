## Objectives
- Identify OPSEC risks

---

## Steps

### 1. Metadata Analysis
Review samples (headers, documents) for:
- Creator metadata  
- Timezone leaks  
- Software version info  
- Language configuration  

### 2. Linguistic Fingerprint Detection
Look for:
- Specific idioms  
- Sentence patterns  
- Repeated punctuation behavior  
- Regional spellings  

### 3. Infrastructure Pattern Recognition
Analyze:
- Domain naming style  
- Hosting provider patterns  
- Timing of network activity  
- Reused TLS cert patterns  

### 4. Attribution Risk Assessment
Write how an attacker could be deanonymized through:
- Behavioral clustering  
- Metadata correlation  
- Infrastructure overlaps

## Deliverables
- OPSEC weakness report  
- Pattern correlation table  
- Attribution-risk summary
